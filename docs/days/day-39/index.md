# 第 39 天：项目讲解稿和简历

::: tip 阶段
**阶段：** 面试 / 简历 / 投递

**用法：** 每个时间段都已经把“具体看哪些章节”和“学完怎么验收”放在一起。先看验收标准，再照着具体安排做。
:::

[[toc]]

## 08:30-09:00 晨间复盘

::: tip 这一段学完要达到
- 知道：项目讲解是面试核心。
- 理解：面试官会顺着你的项目深挖。
- 学会：打开 README、代码、笔记一起准备。
- 验收：列出项目 6 个亮点。
:::

### 具体安排

- 打开 README 和项目。
- 写今天目标：能 5-8 分钟讲项目。


## 09:00-12:00 上午学习

::: tip 这一段学完要达到
- 知道：README、生产部署、构建都是项目完整度的一部分。
- 理解：简历项目要能运行、能展示、能解释。
- 学会：回顾项目工程配置。
- 验收：能说出项目启动、构建、测试命令。
:::

### 具体安排

- 看 GitHub README 指南：https://docs.github.com/zh/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/about-readmes
  - 必看章节：README 的作用、可包含的信息、相对链接和图片。
  - 读完要会：写清项目介绍、技术栈、启动方式、功能截图、亮点。
- 复看 Vue 生产部署：https://cn.vuejs.org/guide/best-practices/production-deployment.html
- 复看 Vite 构建：https://cn.vitejs.dev/guide/build.html


## 12:00-14:00 中午轻复盘

::: tip 这一段学完要达到
- 知道：项目亮点不能只写“用了 Vue3”。
- 理解：要写你解决了什么问题。
- 学会：提炼亮点。
- 验收：写出 TS 类型设计、RBAC、组件封装、请求封装、性能、测试 6 个亮点。
:::

### 具体安排

- 整理项目亮点：
  - TS 类型设计
  - RBAC 权限
  - 组件封装
  - 请求封装
  - 性能优化
  - 测试


## 14:00-18:00 下午项目

::: tip 这一段学完要达到
- 知道：项目讲解要有结构。
- 理解：背景 -&gt; 技术栈 -&gt; 难点 -&gt; 方案 -&gt; 结果。
- 学会：写项目讲解稿和简历项目描述。
- 验收：讲项目能控制在 5-8 分钟。
:::

### 具体安排

- 写项目讲解稿。
- 写一页简历。
- 项目描述按“背景 + 技术 + 难点 + 结果”写。


## 19:30-21:00 晚上算法

::: tip 这一段学完要达到
- 知道：笔试仍然要练。
- 理解：项目强但算法完全不会也会卡。
- 学会：限时刷题。
- 验收：记录 60 分钟模拟结果。
:::

### 具体安排

- LeetCode 模拟 60 分钟。
  - 做题步骤：先读题目描述、示例、约束；自己写 20-30 分钟；能 AC 后再看题解优化。
  - 提交后要补：时间复杂度、空间复杂度、一个容易错的边界条件。


## 21:00-22:30 夜间输出

::: tip 这一段学完要达到
- 知道：录音能检查表达是否自然。
- 理解：讲项目不能像念 README。
- 学会：录音并修改讲稿。
- 验收：完成简历 v1 + 项目讲解稿。
:::

### 具体安排

- 录音讲项目一次。
- 修改讲解稿。
- 当天产出：简历 v1 + 项目讲解稿。
