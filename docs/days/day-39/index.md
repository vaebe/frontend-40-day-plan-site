# 第 39 天：项目讲解稿和简历

::: tip 阶段
**阶段：** 面试 / 简历 / 投递

**用法：** 先看“今日安排”，做完后用“学习结果和验收”检查自己是否真的学会。
:::

[[toc]]

## 今日安排

08:30-09:00：

- 打开 README 和项目。
- 写今天目标：能 5-8 分钟讲项目。

09:00-12:00：

- 看 GitHub README 指南：https://docs.github.com/zh/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/about-readmes
- 复看 Vue 生产部署：https://cn.vuejs.org/guide/best-practices/production-deployment.html
- 复看 Vite 构建：https://cn.vitejs.dev/guide/build.html

12:00-14:00：

- 整理项目亮点：
  - TS 类型设计
  - RBAC 权限
  - 组件封装
  - 请求封装
  - 性能优化
  - 测试

14:00-18:00：

- 写项目讲解稿。
- 写一页简历。
- 项目描述按“背景 + 技术 + 难点 + 结果”写。

19:30-21:00：

- LeetCode 模拟 60 分钟。

21:00-22:30：

- 录音讲项目一次。
- 修改讲解稿。
- 当天产出：简历 v1 + 项目讲解稿。

## 学习结果和验收

08:30-09:00：

- 知道：项目讲解是面试核心。
- 理解：面试官会顺着你的项目深挖。
- 学会：打开 README、代码、笔记一起准备。
- 验收：列出项目 6 个亮点。

09:00-12:00：

- 知道：README、生产部署、构建都是项目完整度的一部分。
- 理解：简历项目要能运行、能展示、能解释。
- 学会：回顾项目工程配置。
- 验收：能说出项目启动、构建、测试命令。

12:00-14:00：

- 知道：项目亮点不能只写“用了 Vue3”。
- 理解：要写你解决了什么问题。
- 学会：提炼亮点。
- 验收：写出 TS 类型设计、RBAC、组件封装、请求封装、性能、测试 6 个亮点。

14:00-18:00：

- 知道：项目讲解要有结构。
- 理解：背景 -&gt; 技术栈 -&gt; 难点 -&gt; 方案 -&gt; 结果。
- 学会：写项目讲解稿和简历项目描述。
- 验收：讲项目能控制在 5-8 分钟。

19:30-21:00：

- 知道：笔试仍然要练。
- 理解：项目强但算法完全不会也会卡。
- 学会：限时刷题。
- 验收：记录 60 分钟模拟结果。

21:00-22:30：

- 知道：录音能检查表达是否自然。
- 理解：讲项目不能像念 README。
- 学会：录音并修改讲稿。
- 验收：完成简历 v1 + 项目讲解稿。
