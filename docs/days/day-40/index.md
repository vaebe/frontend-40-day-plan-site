## 每天最终自检
每天睡前问自己 6 个问题：
```text
1. 今天上午的概念，我能不能不看文档讲出来？
2. 今天下午的项目功能，我能不能打开页面演示？
3. 今天的 TS/JS 知识，我有没有写过代码例子？
4. 今天的算法题，我能不能说出思路和复杂度？
5. 今天有没有 Git commit？
6. 今天最不懂的问题是什么，明天第一件事要不要补？
```
## 阶段能力标准
第 1-8 天结束后：
- 你应该知道：Vite 项目怎么跑，Vue 项目目录怎么分，JS 作用域、闭包、Promise、事件循环是什么。
- 你应该学会：改页面、写基础工具函数、处理登录成功/失败状态、写简单算法。
- 你应该产出：项目能启动，登录页和 Dashboard 有你自己的改动，JS 基础笔记成型。
第 9-16 天结束后：
- 你应该知道：TS 基础类型、interface、联合类型、类型收窄、泛型、工具类型是什么。
- 你应该学会：给业务对象、接口返回、表单状态写类型。
- 你应该产出：项目核心类型文件、typed utils、typed mock API、TS 复盘笔记。
第 17-29 天结束后：
- 你应该知道：Vue3 Composition API、Router、Pinia、Axios、RBAC、组件封装、ECharts、composable 怎么串成项目。
- 你应该学会：做业务页、封装 SearchForm/DataTable、做权限控制、做看板。
- 你应该产出：一个有后台系统雏形的 Vue3 + TS 项目。
第 30-34 天结束后：
- 你应该知道：浏览器渲染、HTTP、缓存、跨域、安全、测试、部署的基本面试点。
- 你应该学会：用 DevTools 排查问题，写单测，完善 README。
- 你应该产出：性能/网络/安全/测试/部署文档和测试用例。
第 35-40 天结束后：
- 你应该知道：自己的项目亮点和短板。
- 你应该学会：讲项目、答基础题、写简历、模拟面试。
- 你应该产出：可投递简历、项目讲解稿、面试问答笔记、投递清单。
:::

## 没学过 TS 的特别提醒

1. 第 1-8 天不要因为看不懂类型而焦虑，先把项目跑起来。
2. 第 9-16 天每天都要打开 TypeScript Playground，光看文档不够。
  - 练习方式：把上午每个类型知识点都写成 1 个最小例子，故意写错一次看报错。
  - 读完要会：能根据报错定位是哪一个变量、参数或返回值类型不对。
3. 不要一开始学类型体操，大厂实习更看你能不能把业务类型写清楚。
4. 项目里尽量少写 `any`，但初学时遇到卡住可以先临时写，晚上复盘再改。
5. TS 学习顺序就是：基础类型 -&gt; interface -&gt; 联合类型 -&gt; 类型收窄 -&gt; 泛型 -&gt; 工具类型 -&gt; Vue TS。

你这版 40 天的目标不是“精通 TypeScript”，而是达到：能看懂 Vue3 + TS 项目，能给业务数据、接口返回、组件 props、表单状态写清楚类型，并且面试时讲得明白。
