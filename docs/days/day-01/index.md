### 必做 3：完成第一次固定 DevTools 排错（80 分钟）

**修改或创建这些文件**

- `campus-activity-board/index.html`
- `campus-activity-board/src/app.js`
- `notes/day-01.md`

**按顺序执行**

1. starter 在 index.html 中故意把脚本路径写成 ./src/main.js，但真实文件是 ./src/app.js；刷新先观察页面静态内容仍存在。
2. 依次查看 Console 报错、Network 中失败的 main.js、index.html 的 script 行；把三条证据写入笔记。
3. 只修改 script src 为 ./src/app.js，刷新确认 Network 200 且 Console 出现“app ready”。
4. 在 app.js 第一行设置断点，刷新并确认命中；只记录当前文件名和一项可见变量，然后继续执行。
5. 按“现象 → 证据 → 猜测 → 单点修改 → 回归”写完整排错记录。

**完成后应该看到**

- 能依据 404 找到错误路径
- 修复后 Console 显示 app ready
- 排错记录不是一句“路径错了”

**立即测试，不要留到晚上**

- [ ] 学习者自行把 app.js 临时改成 appx.js，按相同步骤复现并恢复，随后立即确认 git diff
- [ ] 修复后刷新三次仍无红色错误
