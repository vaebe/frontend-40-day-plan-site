# 第 35 天：React 快速了解，别深陷

::: tip 阶段
**阶段：** 面试 / 简历 / 投递

**用法：** 每个时间段都已经把“具体看哪些章节”和“学完怎么验收”放在一起。先看验收标准，再照着具体安排做。
:::

[[toc]]

## 08:30-09:00 晨间复盘

::: tip 这一段学完要达到
- 知道：今天只是了解 React，不转移主线。
- 理解：大厂可能问 React，但你主项目仍然是 Vue。
- 学会：定好边界。
- 验收：写下“今天不重做项目”。
:::

### 具体安排

- 写今天目标：能看懂基础 React，不把主线带偏。


## 09:00-12:00 上午学习

::: tip 这一段学完要达到
- 知道：React 有 JSX、props、state、组件。
- 理解：React 更强调用 JS 描述 UI。
- 学会：看懂简单 React 组件。
- 验收：能写一个最小 React 组件。
:::

### 具体安排

- 看 React Learn：https://react.dev/learn
  - 只看总览：Quick Start、Thinking in React 的入口，不要今天深挖生态。
  - 读完要会：知道 React 是组件化 UI 库，和 Vue 都围绕状态驱动视图。
- 看 描述 UI：https://react.dev/learn/describing-the-ui
  - 必看章节：Your First Component、Importing and Exporting Components、Writing Markup with JSX、Passing Props。
  - 读完要会：能看懂 React 组件和 JSX。
- 看 添加交互：https://react.dev/learn/adding-interactivity
  - 必看章节：Responding to Events、State: A Component's Memory、Render and Commit。
  - 读完要会：知道 React 用 state 触发重新渲染。
- 看 状态管理：https://react.dev/learn/managing-state
  - 必看章节：Reacting to Input with State、Choosing the State Structure、Sharing State Between Components。
  - 读完要会：能对比 Vue props/emit 和 React props/state。


## 12:00-14:00 中午轻复盘

::: tip 这一段学完要达到
- 知道：React 和 Vue 都是组件化框架。
- 理解：两者状态更新和模板表达方式不同。
- 学会：写基础对比。
- 验收：列出 5 条 Vue/React 差异。
:::

### 具体安排

- 看 Thinking in React：https://react.dev/learn/thinking-in-react
  - 必看全部章节：拆组件层级、找最小 state、数据单向流动。
  - 读完要会：能用 React 思路拆一个列表搜索页面。
- 写 Vue 和 React 差异初稿。
