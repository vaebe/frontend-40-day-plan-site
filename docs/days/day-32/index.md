## 学习结果和验收

08:30-09:00：

- 知道：安全不是后端一个人的事。
- 理解：前端也可能引入 XSS 风险。
- 学会：搜索项目里的 `v-html`。
- 验收：知道项目是否存在危险 HTML 渲染。

09:00-12:00：

- 知道：XSS、CSRF、CSP、输入校验。
- 理解：XSS 是注入脚本，CSRF 是借用户身份发请求。
- 学会：说出基础防御方式。
- 验收：能举例说明 XSS 如何发生。

12:00-14:00：

- 知道：用户输入是安全风险入口。
- 理解：展示用户输入时要谨慎。
- 学会：列输入输出点。
- 验收：列出项目 3 个输入点。

14:00-18:00：

- 知道：表单限制可以减少错误输入。
- 理解：前端校验提升体验，但后端也必须校验。
- 学会：加长度限制和基础校验。
- 验收：README 有安全检查清单。

19:30-21:00：

- 知道：错题要归类。
- 理解：数组、树、回溯、dp 的错因不同。
- 学会：给错题打标签。
- 验收：2 道错题完成复盘。

21:00-22:30：

- 知道：XSS 和 CSRF 是高频面试题。
- 理解：两者攻击方式不同，防御也不同。
- 学会：写安全面试答案。
- 验收：写完《XSS 和 CSRF 区别》。
