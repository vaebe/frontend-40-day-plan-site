# 第 32 天：前端安全

::: tip 阶段
**阶段：** 浏览器 / 网络 / 性能 / 测试 / 部署

**用法：** 每个时间段都已经把“具体看哪些章节”和“学完怎么验收”放在一起。先看验收标准，再照着具体安排做。
:::

[[toc]]

## 08:30-09:00 晨间复盘

::: tip 这一段学完要达到
- 知道：安全不是后端一个人的事。
- 理解：前端也可能引入 XSS 风险。
- 学会：搜索项目里的 `v-html`。
- 验收：知道项目是否存在危险 HTML 渲染。
:::

### 具体安排

- 搜项目里有没有 `v-html`。
- 写今天目标：知道 XSS 和 CSRF。


## 09:00-12:00 上午学习

::: tip 这一段学完要达到
- 知道：XSS、CSRF、CSP、输入校验。
- 理解：XSS 是注入脚本，CSRF 是借用户身份发请求。
- 学会：说出基础防御方式。
- 验收：能举例说明 XSS 如何发生。
:::

### 具体安排

- 看 MDN Web 安全：https://developer.mozilla.org/zh-CN/docs/Web/Security
  - 必看主题：同源策略、CORS、CSP、HTTPS、常见攻击概念。
  - 读完要会：知道 XSS、CSRF、点击劫持分别大概是什么。
- 看 Vue 安全：https://cn.vuejs.org/guide/best-practices/security.html
  - 必看章节：报告安全漏洞、规则 1：不要使用非可信模板、潜在危险、最佳实践。
  - 读完要会：知道为什么不能把用户输入当模板编译。
- 看 CSP：https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Guides/CSP
  - 必看章节：威胁、配置 CSP、示例、测试策略。
  - 读完要会：知道 CSP 用来限制资源来源，降低 XSS 风险。


## 12:00-14:00 中午轻复盘

::: tip 这一段学完要达到
- 知道：用户输入是安全风险入口。
- 理解：展示用户输入时要谨慎。
- 学会：列输入输出点。
- 验收：列出项目 3 个输入点。
:::

### 具体安排

- 整理：哪些输入来自用户，哪些输出会渲染到页面。


## 14:00-18:00 下午项目

::: tip 这一段学完要达到
- 知道：表单限制可以减少错误输入。
- 理解：前端校验提升体验，但后端也必须校验。
- 学会：加长度限制和基础校验。
- 验收：README 有安全检查清单。
:::

### 具体安排

- 给表单输入加基础长度限制。
- 避免危险 HTML 注入。
- README 写安全设计小节。
